Jobs & Exporters

Jobs and Exporters expose system and service metrics in a plain text HTTP format. Exporters are agents deployed alongside apps to translate third-party metrics (like database stats or hardware usage) to Prometheus format.

Deployment

Install Prometheus

Prometheus is compiled as a single binary and has no external dependencies. Learn how to configure and install Prometheus in your environment.

1

Download and Extract Binary

Download the Linux binary using wget and unpack it.

2

Configure Directories and User

Create a system user and set up configuration and data directories with appropriate permissions.

3

Deploy systemd Service

Create a systemd unit file to manage the Prometheus process and enable auto-start on boot.

install-linux.sh
$# Download tarball
$wget https://github.com/prometheus/prometheus/releases/download/v2.53.0/prometheus-2.53.0.linux-amd64.tar.gz
$tar -xvf prometheus-2.53.0.linux-amd64.tar.gz
$# Create system user & directories
$sudo useradd --no-create-home --shell /bin/false prometheus
$sudo mkdir /etc/prometheus /var/lib/prometheus
$# Copy binaries and configs
$sudo cp prometheus-2.53.0.linux-amd64/prometheus /usr/local/bin/
$sudo cp prometheus-2.53.0.linux-amd64/promtool /usr/local/bin/
$sudo cp -r prometheus-2.53.0.linux-amd64/consoles /etc/prometheus
$sudo cp -r prometheus-2.53.0.linux-amd64/console_libraries /etc/prometheus
$sudo cp prometheus-2.53.0.linux-amd64/prometheus.yml /etc/prometheus/
$# Change ownership
$sudo chown -R prometheus:prometheus /etc/prometheus /var/lib/prometheus
$# Run Prometheus Server
$prometheus --config.file=/etc/prometheus/prometheus.yml
1

Install via Homebrew

Run the Homebrew install command to download and set up Prometheus natively.

2

Locate Configurations

Configurations are stored at /opt/homebrew/etc/prometheus.yml on Apple Silicon.

3

Start the Daemon

Manage the service state and run it in the background using homebrew services.

macos-brew.sh
%# Install using Homebrew
%brew install prometheus
%# Start background daemon
%brew services start prometheus
%# Or run manually in foreground:
%prometheus --config.file=/opt/homebrew/etc/prometheus.yml
1

Using Package Managers

Easily install with Chocolatey (choco) or Scoop.

2

Direct Binary Execution

Download the zip package, extract it, and execute prometheus.exe directly inside Powershell.

3

Run in Terminal

By default, Prometheus launches at localhost:9090.

windows-cmd.ps1
PS># Install with Chocolatey:
PS>choco install prometheus
PS># Or run extracted binary directly:
PS>cd C:\prometheus-folder
PS>.\prometheus.exe --config.file="prometheus.yml"

Define prometheus.yml

Create a local config file mapping scraper targets to localhost.

2

Run Container

Bind port 9090 and mount the local configuration file to the container path.

3

Access and Verify

Visit container dashboard, query raw container statistics, or view targets.

docker-compose.yml

$docker run -d --name prometheus -p 9090:9090 -v /path/to/prometheus.yml:/etc/prometheus/prometheus.yml prom/prometheus
Instrumentation

The Four Metric Types

Prometheus classifies client metrics into four distinct types. Click the action deck below to simulate app behavior and watch the raw output generate in real-time.

Counter

A cumulative metric that only goes up or resets to 0 on restart. Ideal for counting requests, errors, or completed tasks.

http_requests_total

Gauge

Represents a single numerical value that can go up and down. Perfect for CPU usage, memory, or queue sizes.

node_memory_usage_bytes

Histogram

Samples observations (e.g. durations) and counts them in configurable buckets. Provides sum and count metrics.

http_request_duration_seconds_bucket

Summary

Similar to a histogram, it calculates configurable quantiles (like 99th percentile) over a sliding time window.

http_request_duration_seconds{quantile="0.99"}
